# Vision Phase — VP1: MVP

This directory captures the product vision for the **Minimum Viable Product** (Phase 1).

Everything here maps to **Theme TH1** in `docs/themes/TH1-*/`.

## What to capture here

Create markdown files covering any or all of:

- **problem.md** — What pain point does this solve?
- **users.md** — Who are the target users/personas?
- **features.md** — Core features this phase must deliver
- **success-criteria.md** — How do we know VP1 is done?
- **constraints.md** — Technology, budget, timeline, compliance constraints
- **open-questions.md** — Unknowns to resolve before implementation
- **brainstorm.md** — Free-form notes, sketches, ideas

There is no rigid structure — this is a creative canvas. The **architect** and **product-owner** agents will read these files and produce structured architecture and user stories from them.

## Tips

- Be specific about user problems, vague about solutions (let the architect decide)
- Include examples and scenarios — they become BDD test cases later
- Capture non-functional requirements early (performance, security, scale)
- Mark open questions explicitly — they'll be resolved during the architecture phase
