# Documentation Layout

The documentation tree is organized around four active concerns:

| Path | Purpose |
| --- | --- |
| `docs/vision_of_product/` | Vision phases and product intent |
| `docs/architecture/` | Architecture overview, design docs, and setup |
| `docs/architecture/adrs/` | Architecture decision records |
| `docs/themes/` | Themes, epics, and user stories |
| `docs/plan/` | Backlog state, changelog, session log, and release notes |

Historical evaluation material remains in `docs/experiments/`.
